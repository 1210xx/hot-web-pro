JavaScript 事件循环机制

可视化演示 JavaScript 的并发模型如何工作

什么是事件循环？

事件循环是 JavaScript 的并发模型的核心机制，它允许 JavaScript 在执行异步操作时不会阻塞主线程。

关键组成部分

  • 调用栈 (Call Stack)：跟踪当前正在执行的函数
  • Web APIs：浏览器提供的异步功能（setTimeout, DOM事件, AJAX等）
  • 回调队列 (Callback Queue)：存放已完成的异步操作的回调函数
  • 微任务队列 (Microtask Queue)：存放Promise回调等微任务

工作原理

1. JavaScript 依次执行同步代码，将其添加到调用栈

2. 遇到异步操作时，将其交给 Web APIs 处理

3. Web APIs 完成后，将回调函数放入相应的队列

4. 当调用栈为空时，事件循环先处理微任务队列中的所有任务

5. 然后从回调队列中取出一个任务执行

6. 重复此过程，形成事件循环

console.log('开始');

setTimeout(() => {
  console.log('setTimeout 回调');
}, 0);

Promise.resolve().then(() => {
  console.log('Promise 回调');
});

console.log('结束');

// 输出顺序:
// 开始
// 结束
// Promise 回调
// setTimeout 回调
